AGENTE GENERADOR DE IMAGENES
Version exportable del flujo de Retiru

OBJETIVO

Este documento describe exactamente el flujo que ha dado el salto de calidad en Retiru para generar portadas realmente fotograficas y no "dibujitos" o renders con look IA.

La idea es que puedas llevarte este archivo a otro proyecto y usarlo como:

1. especificacion de producto
2. guia de implementacion
3. prompt pack para un agente
4. manual para reproducir el mismo sistema con otros contenidos: fichas de producto, articulos del blog, colecciones, landings, destinos, etc.


RESUMEN EJECUTIVO

La mejora no vino solo por cambiar el modelo de imagen.

La mejora real vino de combinar tres cosas:

1. Un DOSSIER rico y bien estructurado a partir de la ficha original.
2. Dos pasadas de GPT-5.4:
   - una para sintetizar la ficha en un prompt visual
   - otra para quitar el look IA y empujarlo hacia fotografia real
3. Un generador de imagen configurado con gpt-image-2 y parametros compatibles con formato panoramico y calidad alta.

El sistema funciona porque NO se le pide al modelo de imagen "hazme una portada bonita".
Se le entrega antes una interpretacion editorial, geografica, tematica y fotografica de la ficha.


ARQUITECTURA DEL FLUJO

FLUJO COMPLETO

Entrada estructurada
-> construir dossier textual rico
-> GPT-5.4 Prompt Builder
-> GPT-5.4 Realism Refiner
-> concatenar coletilla final de realismo fotografico
-> gpt-image-2 (salida raster alta calidad en API, p. ej. PNG)
-> (Furgocasa) reencode WebP con sharp y subir a Storage
-> guardar URL en CMS


MODELOS Y PARAMETROS USADOS

Modelo para interpretar la ficha y redactar prompts:
- gpt-5.4

Primera pasada:
- temperature: 0.32
- max_tokens: 900

Segunda pasada:
- temperature: 0.18
- max_tokens: 900

Modelo de imagen:
- gpt-image-2

Parametros de generacion de imagen:
- n: 1
- size: 1536x1024
- quality: high
- output_format: png (en la llamada a OpenAI; asi el modelo entrega maxima calidad de pixel)

Publicacion en Furgocasa (despues de recibir el PNG):
- sharp: conversion a WebP (calidad por defecto 85; env BLOG_COVER_WEBP_QUALITY 1-100)
- Supabase Storage: bucket blog, ruta ai-covers/{slug}-{timestamp}.webp, contentType image/webp
- La web sigue usando next/image; el origen WebP reduce peso en CDN y enlaces directos

Notas:
- Antes se usaba DALL-E 3. Este flujo fue migrado primero a gpt-image-1.5 y actualmente usa gpt-image-2.
- En Furgocasa, la combinacion actual es:
  GPT-5.4 para pensar la escena + gpt-image-2 para ejecutarla + WebP al publicar.

Referencias visuales (Furgocasa):
- Carpeta images/IA_blog/ mas rutas fijas de vehiculos reales; se normalizan con sharp (JPEG limpio, tamano acotado) antes de images.edit.
- Si el lote falla, se prueba referencia individual; si todo falla, solo prompt.
- Regla de producto en prompts: ninguna camper con dos toldos; si hay toldo, uno solo en lateral derecho, como en fotos reales.
- Diversidad: las referencias definen morfologia del vehiculo, no el encuadre; angulo y distancia los dicta el articulo.

Scripts npm en el repo:
- npm run generate:blog-cover -- "https://www.furgocasa.com/es/blog/..."
- npm run reencode:blog-cover-webp -- "url1" "url2"   (solo re-WebP sin IA; en Windows alternativa: npx tsx scripts/generate-blog-cover.ts reencode-webp ...)
- npm run generate:blog-body-images -- "https://..."                             (imagenes para el cuerpo; gemelo del de portadas)
- npm run generate:blog-cover-and-body -- "https://..."                          (portada + cuerpo en un solo comando, comparten modelo de vehiculo)


AGENTE DE IMAGENES DE CUERPO (may 2026)

Pipeline gemelo, especifico para insertar 2-3 imagenes dentro del articulo, despues de los <h2> elegidos por la IA.

Modulo: src/lib/blog/generate-blog-body-images.ts

Decisiones del agente (en orden):
1. Detecta los <h2> del HTML del post (regex tolerante <h2 ...>...</h2>) y los numera con anchor_index 0..N-1.
2. Calcula TARGET_IMAGE_COUNT segun longitud:
   - <800 palabras -> 2
   - >=800 palabras -> 3
3. PLANNER (gpt-5.4 con response_format json_object):
   - Recibe DOSSIER del articulo + lista de H2 + DECISION_VEHICULO_GLOBAL (modelo de la portada) + TARGET_IMAGE_COUNT.
   - Devuelve items[] con: anchor_index, include_vehicle, section_focus, alt_es, caption_es, draft_prompt.
   - Reglas duras: no repetir anchor_index; no proponer dos H2 consecutivos si se puede evitar; en articulos de >=4 secciones, normalmente saltarse H2 0 para no competir con la portada; al menos 1 imagen con camper si el articulo es de viajes/rutas, salvo que sea solo paisaje.
4. REFINER (gpt-5.4): por cada item, reescribe el draft_prompt como parrafo "Fotografia hiperrealista y cinematografica de..." en espanol. Se le pasa COHERENCIA_PORTADA (resumen del prompt usado en portada) para forzar diferenciacion en encuadre, hora del dia, atmosfera o tipo de plano.
5. Generacion (gpt-image-2, 1536x1024, quality high):
   - Si include_vehicle=true: openai.images.edit con las referencias del modelo (mismo que portada). Coletilla VEHICLE_REFERENCE_TAIL recordando "nunca dos toldos" y que las referencias definen el VEHICULO, no el encuadre.
   - Si include_vehicle=false: openai.images.generate. Coletilla IMAGE_REALISM_TAIL.
6. WebP con sharp -> Supabase Storage en blog/ai-body/{slug}-{n}-{timestamp}.webp.
7. Inyeccion en posts.content:
   - Limpia toda <figure data-ai-body-image="1" ...>...</figure> previa (idempotencia).
   - Inserta despues del cierre </h2> del anchor_index correspondiente:
     <figure data-ai-body-image="1" data-anchor="<slug-del-h2>"><img src="..." alt="..." loading="lazy" /><figcaption>...</figcaption></figure>
8. Persiste manifiesto en posts.images.ai_body (campo Json ya existente; no toca schema):
   {
     version: 1,
     vehicle_model_key, vehicle_model_label, generated_at,
     items: [ { anchor_slug, anchor_index, anchor_text, position: "after_h2", url, storage_path, alt_es, caption_es, include_vehicle, vehicle_model_key, vehicle_model_label, prompt, generated_at } ]
   }

Variables de entorno:
- BLOG_BODY_TEXT_MODEL (default gpt-5.4)
- BLOG_BODY_IMAGE_MODEL (default gpt-image-2)
- BLOG_BODY_WEBP_QUALITY (1-100, default 85)
- BLOG_BODY_USE_VEHICLE_REFERENCES=false para desactivar referencias visuales (prompt-only)

Idempotencia y regeneracion:
- Si posts.images.ai_body ya tiene items, el script sale con skippedReason. Hay que pasar --force para regenerar.
- Cualquier regeneracion limpia primero las <figure data-ai-body-image="1"> existentes y reinserta las nuevas.

Sintaxis CLI:
- Usar SIEMPRE --clave=valor (npm se traga los flags que van separados con espacio).
  Bien:  npm run generate:blog-body-images -- "URL" --vehicle=adria-twin --max-images=2
  Mal:   npm run generate:blog-body-images -- "URL" --vehicle adria-twin


OBJETIVO VISUAL

El objetivo visual no es "bonito", ni "artistico", ni "instagramable".

El objetivo visual exacto es:

- fotografia editorial hiperrealista
- luz creible
- materiales concretos
- atmosfera real
- composicion premium
- nada de ilustracion
- nada de glow fantasioso
- nada de piel plastica
- nada de simetria postalera de IA

Frase de control mental:

"Tiene que parecer una fotografia tomada por un fotografo profesional en un lugar real, no una imagen generada."


POR QUE FUNCIONA

El error habitual en muchos flujos es este:

1. coger un titulo y un resumen
2. mandar eso directo al modelo de imagen
3. pedir "realistic, cinematic, premium"

Eso suele dar:

- escenas genericas
- composiciones irreales
- exceso de color
- personas plasticas
- vibe de render o cartel

En cambio, este flujo fuerza al sistema a:

1. leer toda la ficha
2. decidir que escena SI representa de verdad el contenido
3. concretar geografia, estacion, actividad, materiales, publico y luz
4. pulir la escena para quitarle "look IA"

PASO 1 - CONSTRUIR EL DOSSIER

REGLA CLAVE:
No enviar campos sueltos al LLM. Hay que componer un DOSSIER narrativo bien rotulado.

El dossier debe:

- juntar toda la informacion relevante
- recortar lo redundante
- conservar datos concretos
- separar bien bloques tematicos

En Retiru se hizo asi:

1. Titulo
2. Resumen
3. Descripcion larga
4. Version inglesa si existe
5. Geografia, calendario y clima ambiente
6. Categorias
7. Incluye
8. No incluye
9. Programa
10. Idiomas


HEURISTICAS IMPORTANTES DEL DOSSIER

1. Limitar longitud de descripcion
- descripcion larga: max 2800 caracteres
- programa: max 1200 caracteres

2. Inferir estacion desde la fecha
- marzo-mayo -> primavera
- junio-agosto -> verano
- septiembre-noviembre -> otono
- diciembre-febrero -> invierno

3. Calcular duracion
- si hay start_date y end_date, convertirlo en "noche(s) de experiencia"

4. Convertir arrays a texto util
- categorias en linea
- incluye como lista separada por "; "
- programa resumido por dias

5. Marcar "no incluye"
- no como bloqueo duro del modelo de imagen
- sino como advertencia de que no debe ser protagonista

PASO 2 - PROMPT BUILDER

OBJETIVO

La primera pasada NO genera la imagen.
Genera un prompt ya muy bueno, pero todavia con riesgo de sonar demasiado "bonito".

La funcion de este prompt builder es:

- elegir la escena correcta
- fijar geografia y actividad real
- elegir luz y momento del dia
- introducir texturas concretas
- evitar genericidad


PROMPT BUILDER ORIGINAL DE RETIRU

Usa esta version si quieres replicar la logica exacta:

Eres un agente senior: director de arte + location scout + especialista en prompts para generacion de imagenes fotorrealistas. Recibes un DOSSIER COMPLETO de un retiro (geografia, fechas, categorias, textos, programa, incluidos). Tu UNICA salida es UN parrafo en espanol que el modelo de imagen usara tal cual: debe ser la mejor posible.

ANTES de escribir (mentalmente, no lo imprimas): (1) Elige el escenario visual mas especifico y honesto con el dossier, no un "paraiso generico". (2) Conecta TEMATICA: yoga/meditacion/ceramica/ayurveda/circulo/mar/desierto/montana segun categorias + descripcion + programa + incluidos. (3) Elige UNA luz creible de dia (manana luminosa, media manana, tarde clara o golden hour todavia alta) coherente con estacion y region si el dossier lo permite. (4) Anade 2-4 sustantivos CONCRETOS de textura o material (adobe, lino, arcilla, corcho, sal, musgo, dunas, olivos) alineados con el lugar y la actividad, no adjetivos vacios. (5) Si hay conflicto entre campos, prima descripcion + titulo + destino sobre suposiciones. (6) Piensa como si un fotografo profesional estuviera fisicamente alli con una camara full frame de alta gama haciendo una foto real para una portada editorial, no como si estuviera "creando arte".

REGLAS DURAS:
- Geografia: respeta destino/direccion/fechas/estacion inferida; no inventes monumentos ni ciudades nombradas si no salen en el dossier.
- Luz/horario: PROHIBIDO noche, anochecer oscuro, hora azul, sol ya puesto, amanecer antes de salir el sol o escenas subexpuestas. La foto debe sentirse tomada con luz natural real entre las 09:00 y las 20:30 de horario de verano del sur de Espana: luminosa, clara y usable como portada.
- Publico: si el texto indica solo mujeres u otro colectivo, respetalo; si no, grupo mixto o sin genero explicito, sin estereotipos.
- Personas: como mucho pocas figuras; rostros sin identidad (espaldas, lejania, desenfoque, silueta). Nunca celebridades. Si la presencia humana hace la imagen menos realista, prioriza una escena de entorno real con presencia humana minima, lejana o incluso ninguna.
- Prohibido en la escena: texto legible, logotipos, carteles, moviles como foco, marcas, datos de contacto.
- Evita "look IA": cielos neon, piel de plastico, simetria de postal barata, oversaturacion, HDR agresivo, tonos morados irreales, niebla magica, render/pintura/ilustracion, perfeccion plastica, piel suavizada, brillo artificial, composicion imposible o escenografia de fantasia, sombras excesivamente cerradas o escenas tenebrosas.

FORMATO DE SALIDA (obligatorio):
- Exactamente UN parrafo en espanol, sin saltos de linea, sin vinetas, sin comillas, sin markdown.
- Entre 400 y 1100 caracteres: denso y cinematografico, pero fotografico (no guion de pelicula).
- Debe empezar con: Fotografia hiperrealista y cinematografica de
- Debe terminar integrando (en la misma frase final o penultima) la idea de: composicion editorial premium, fotografia de viaje y bienestar de lujo, profundidad de campo natural, texturas realistas, encuadre horizontal amplio, sin texto ni logos ni ilustracion, realismo fotografico absoluto, portada web de alta conversion.
- La redaccion debe sonar a encargo fotografico real: camara profesional, luz existente, color natural, detalles imperfectos creibles y atmosfera autentica.

No escribas nada antes ni despues del parrafo.

PASO 3 - REALISM REFINER

OBJETIVO

La segunda pasada es la pieza decisiva.

No inventa el prompt desde cero.
Hace de editor fotografico obsesionado con quitar:

- exceso de belleza artificial
- teatralidad
- postureo de anuncio
- fake luxury
- turismo de folleto
- wellness de Pinterest


PROMPT REALISM REFINER ORIGINAL

Eres un editor fotografico obsesionado con el hiperrealismo. Recibiras:
1) un DOSSIER del evento
2) un primer prompt ya redactado

Tu tarea es REESCRIBIR ese prompt para que parezca todavia mas una fotografia real tomada por un fotografo profesional en localizacion real.

Prioridades:
- La imagen debe parecer una FOTO REAL, no arte generativo.
- Si el borrador suena demasiado bonito, demasiado escenificado, demasiado "wellness instagram", demasiado simetrico o demasiado turistico, rebajalo.
- Da prioridad a entorno real, luz existente, materiales concretos, imperfecciones creibles, composicion editorial contenida.
- Si las personas no son imprescindibles para comunicar el retiro, reduce su protagonismo o dejalas lejanas/secundarias. Si aparecen, nada de poses artificiales ni expresiones de anuncio.
- Corrige cualquier tendencia a oscuridad excesiva: nada de noche, nada de sol escondido, nada de cielos dramaticos oscuros; debe sentirse una franja luminosa y comercialmente util de dia.
- Evita cualquier sensacion de fantasia, exceso de color, teatralidad, glow, piel plastica, postal irreal o escena de catalogo falso.

Reglas:
- Manten coherencia absoluta con el dossier.
- Salida: EXACTAMENTE un parrafo en espanol, sin comillas, sin markdown, sin vinetas, sin saltos de linea.
- Debe empezar por "Fotografia hiperrealista y cinematografica de".
- Debe sonar a encargo fotografico premium real, no a instruccion artistica abstracta.
- No anadas explicaciones sobre lo que has cambiado. Devuelve solo el prompt final.

PASO 4 - COLA FINAL DE REALISMO

Despues de las dos pasadas de GPT-5.4, aun se anade una coletilla final fija.
Esto es importante porque:

- normaliza el tipo de acabado
- obliga a la fotografia a quedarse en terreno realista
- baja el riesgo de que el modelo vuelva a "embellecer" demasiado

COLA FINAL USADA EN RETIRU

Tomada como fotografia real con camara full frame profesional y optica de reportaje de alta calidad, luz existente fisicamente creible, color natural y balance de blancos realista, contraste moderado, grano minimo natural, detalle autentico en piel, telas, arena, piedra, vegetacion o arquitectura segun la escena; siempre de dia, luminosa y clara, nunca nocturna ni sombria, con sensacion de franja util entre 09:00 y 20:30 de verano del sur de Espana; si aparecen personas, secundarias, naturales y no posadas; sin HDR agresivo, sin acabado plastico, sin render 3D, sin pintura digital, sin ilustracion, sin tipografia ni logotipos.

RECOMENDACION

No la elimines.
Si adaptas el sistema a otro proyecto, ajusta solo:

- rango horario
- entorno geografico
- tipo de materiales
- presencia o ausencia de personas


PASO 5 - PETICION FINAL A GPT-IMAGE-1.5

PETICION RECOMENDADA

POST https://api.openai.com/v1/images/generations

JSON:

{
  "model": "gpt-image-2",
  "prompt": "PROMPT_FINAL",
  "n": 1,
  "size": "1536x1024",
  "quality": "high",
  "output_format": "png"
}

REGLAS PRACTICAS

- Usa formato panoramico si la imagen va a cabeceras o cards horizontales.
- Usa una sola imagen por llamada para maximizar control editorial.
- Usa png si luego la vas a guardar y servir tal cual.


PASO 6 - LIMPIEZA Y NORMALIZACION DEL PROMPT

Antes de llamar al modelo de imagen:

1. quitar comillas iniciales o finales
2. colapsar espacios multiples
3. truncar a 4000 caracteres si hace falta
4. validar que el prompt no se haya quedado ridiculamente corto

Regla usada:
- si el prompt final tiene menos de 200 caracteres -> error


PARAMETROS QUE HAN DADO BUEN RESULTADO

Para la interpretacion del dossier:
- gpt-5.4
- temp media-baja

Para el refinado:
- gpt-5.4
- temp baja

Para la imagen:
- gpt-image-2
- quality high
- 1536x1024

Esto da una mezcla buena entre:
- obediencia
- realismo
- composicion editorial


COSAS QUE NO HAY QUE HACER

1. No mandar solo titulo + resumen al generador de imagen.
2. No pedir "ultra realistic cinematic" sin contexto real.
3. No abusar de palabras como:
   - magical
   - dreamy
   - ethereal
   - stunning
   - breathtaking
   - luxurious
   - perfect
4. No dejar que el modelo invente geografia.
5. No dejar que meta logos, texto o moviles.
6. No dejar que el prompt se convierta en una pieza de prosa artistica vaga.
7. No mezclar demasiadas escenas en una sola imagen.

- luz alta de golden hour aun util

3. Composicion concreta
- encuadre horizontal amplio
- escena contenida
- profundidad natural
- pocas figuras
- fondo real

4. Imperfeccion real
- telas arrugadas
- barro real
- grano minimo natural
- sombra creible
- arquitectura vivida


GUIA DE ADAPTACION RAPIDA A OTROS CASOS

CASO A - PRODUCTO DE HOGAR

En el dossier anade:
- estancia donde se usa
- material principal
- acabado
- publico objetivo
- tono de marca

Y cambia mentalmente:
- "evento" por "producto"
- "programa" por "contexto de uso"
- "incluye" por "elementos visibles"


CASO B - COSMETICA

Anade:
- textura del producto
- ingredientes
- ritual de uso
- cuarto de bano / tocador / spa realista

Evita:
- gotas imposibles
- piel plastica
- laboratorios futuristas de anuncio barato


CASO C - ARTICULO DE BLOG

Anade:
- tesis del articulo
- emocion buscada
- escena real que simboliza el contenido

Evita:
- metaforas obvias
- montajes demasiado conceptuales


CASO D - FICHA DE HOTEL / VIAJE / EXPERIENCIA

Muy parecido a Retiru.
Es el caso donde este flujo mejor encaja.


PSEUDOCODIGO DE IMPLEMENTACION

function buildDossier(input):
  - recorta descripcion
  - infiere estacion si hay fecha
  - calcula duracion si aplica
  - resume programa / uso / narrativa
  - compone bloques rotulados
  - devuelve string unico

function buildPromptWithLLM(dossier):
  - llama a GPT-5.4 con PROMPT_BUILDER_SYSTEM
  - limpia la respuesta
  - llama a GPT-5.4 con PROMPT_REALISM_REFINER_SYSTEM
  - limpia la respuesta
  - concatena IMAGE_REALISM_TAIL
  - valida longitud
  - devuelve prompt final

function generateImage(prompt):
  - llama a /v1/images/generations
  - model: gpt-image-2
  - size: 1536x1024
  - quality: high
  - output_format: png
  - devuelve buffer o url


CHECKLIST DE CALIDAD

Antes de dar una imagen por buena, preguntate:

1. Parece una foto real hecha por una persona?
2. El lugar parece concreto y no inventado?
3. Hay materiales reales visibles?
4. La luz es creible?
5. Se entiende el tema del contenido sin parecer una fantasia?
6. Hay algun rastro de ilustracion o CGI?
7. Hay exceso de pose o de perfeccion plastica?

Si falla algo de eso, vuelve al prompt.


IDEA CLAVE FINAL

El secreto de este sistema no es "usar un modelo mejor".
El secreto es obligar al sistema a pensar como:

- editor visual
- director de arte
- fotografo de localizacion
- revisor anti-look-IA

Primero piensa.
Luego redacta.
Luego refina.
Y solo al final genera.

Eso es lo que convierte una ficha en una portada con aspecto de fotografia real.
